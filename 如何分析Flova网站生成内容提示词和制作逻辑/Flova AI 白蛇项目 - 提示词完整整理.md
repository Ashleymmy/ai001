## 三、提示词的关键特征总结

### 1. 元素引用机制
使用 `[Element_XXX]` 的方式引用预先生成的角色和物品，确保角色形象在所有镜头中保持一致。

**示例**:
- `[Element_YOUNG_SERVANT]` - 年轻仆人
- `[Element_WHITE_SNAKE]` - 白蛇
- `[Element_PRINCESS]` - 公主

### 2. 镜头类型标注
每个视频提示词都明确标注镜头类型和时长：
- **标准叙事镜头** (约 5-6秒) - 用于常规叙事
- **快速切换** (约 3-4秒) - 用于关键转折点
- **特写镜头** - 强调细节和情绪

### 3. 视觉细节描述
提示词包含丰富的视觉细节：
- **光线**: "傍晚的金色阳光透过高窗洒下"
- **质感**: "画面细腻，强调食物的光泽感"
- **氛围**: "充满生机"
- **动态**: "微风吹拂树叶"

### 4. 旁白对齐
每个镜头都有对应的旁白，使用"对齐旁白"来确保视频内容与叙述同步。

### 5. 情绪和动作描述
- **情绪**: "好奇地"、"偷偷"、"惊奇"
- **微表情**: "瞳孔微微放大"、"表情带有惊奇"
- **动作**: "揭开"、"尝了一小口"、"吞下"

### 6. 视觉特效说明
某些镜头包含特殊的视觉效果：
- "鸣叫声在画面中具象化为音符或对话气泡"

## 四、生成流程

### 图片生成流程
1. **角色设计** → 使用高分辨率模型(2K)生成角色原画
2. **起始帧生成** → 基于角色设计生成每个镜头的第一帧(start_img)
3. **保持一致性** → 通过引用机制确保角色形象统一

### 视频生成流程
1. **静态画面** → 先生成静态的起始帧
2. **动态化** → 将静态画面转化为动态视频(final_video)
3. **经济优化** → 使用720p分辨率降低成本
4. **时长控制** → 根据叙事需求调整每个镜头的时长

## 六、提示词编写技巧

```
[镜头类型] + [主体动作] + [场景元素] + [光线/氛围] + [画面质感]
```

### 2. 元素复用
通过 `[Element_XXX]` 引用预生成的角色，避免重复描述，确保一致性。

### 3. 时长标注

### 4. 旁白同步

### 6. 视觉层次
从大场景到细节，从动作到表情，层次分明。

## 七、总结

Flova的提示词系统具有以下特点：

1. **模块化设计**: 角色、场景、动作分离，便于复用和修改
2. **引用机制**: 通过元素引用确保角色一致性
3. **结构化表达**: 镜头类型、时长、动作、氛围等要素清晰

6. **细节丰富**: 包含光线、质感、情绪、动态等多维度描述

这种提示词设计方式既保证了生成质量，又兼顾了成本控制，非常适合短视频和动画制作。
